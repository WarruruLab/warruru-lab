# ADR-001: 서비스 분리

## 상태

Accepted

## 배경

와르르랩은 개발 중 발생하는 문제를 기록하고,
해결 과정과 판단 근거를 자산으로 남기기 위한 프로젝트이다.

프로젝트는 두 개의 서비스로 구성된다.

- DevTalk: 문제 해결 과정의 로그를 생성하는 서비스
- DevLog: 로그를 분석해 트러블슈팅 블로그 초안을 생성하는 서비스

초기 설계 단계에서 두 서비스를 동시에 개발할지,
아니면 순차적으로 개발할지에 대한 판단이 필요했다.

## 결정

DevTalk을 먼저 개발하고,
DevLog는 DevTalk이 실제로 사용 가능한 상태가 된 이후에 개발한다.

## 이유

- 로그가 충분히 쌓이지 않으면 분석과 정리는 의미가 없다.
- DevTalk이 불편하면 기록 자체가 생성되지 않는다.
- 실제 사용하면서 쌓인 로그를 기준으로 DevLog 설계하는 것이 더 현실적이다.

## 결과

- DevTalk을 1~3회차의 핵심 개발 대상으로 설정했다.
- DevLog 로그 구조와 사용 흐름이 검증된 이후 단계로 미뤘다.
