# WarruruLab

와르르랩(WarruruLab)은 개발 중 발생하는 문제를  
**해결 과정(판단의 근거, 시도, 실패, 최종 해결)을 구조화해 기록하고 자산으로 남기기 위한 개인 프로젝트**다.

AI를 참고해 빠르게 디버깅할수록 대화는 쌓이지만, 시간이 지나면  
**무슨 문제가 있었는지 / 어떤 시도는 왜 실패했는지 / 최종 해결은 무엇이었는지**가 흐려지고 휘발되는 경험을 했다.  
와르르랩은 이 문제를 해결하기 위해, **채팅 기반 기록(DevTalk)** 으로 원본 로그를 남기고  
그 로그를 AI가 다시 분석해 **트러블슈팅 블로그 초안(ProblemWarruru)** 으로 정리해주는 흐름을 만든다.

AI와의 대화는 휘발되지만, **판단의 근거는 기록되어야 한다.**

---

## 💡핵심 아이디어

- 문제 하나 = 세션(Session) 하나 (로그인/HTTP 세션이 아닌, **하나의 문제 해결 단위**)
- 해결 여부는 사람이 직접 확정(`Resolved`)
- 해결된 결과뿐 아니라 **실패한 시도와 그 원인까지 함께 기록**
- 성공·실패를 구분해 분석된 기록을 모두 자산으로 활용

와르르랩에서 중요한 것은 “무엇이 맞았는가”가 아니라  
**왜 이 선택을 했는가**다.

---

## 프로젝트 구성

### 💬DevTalk
- 개발자가 실제로 사용하는 **AI 채팅 기반 문제 해결 기록 서비스**
- 내가 직접 **API(Gemini 등)** 를 연결해 채팅 웹을 만들고, 디버깅 과정에서 나오는 질문·코드·에러 로그·실패한 시도를 대화 형태로 남긴다
- 모든 대화 기록은 **백엔드에 로그로 저장**되어, 문제 해결 과정이 휘발되지 않도록 한다
- 문제가 실제로 해결되었다고 판단한 경우에만 사용자가 직접 `Resolved` 처리한다

→ 문제 해결 과정의 **원본 기록 저장소 (Raw Log)**

### 📝ProblemWarruru
- DevTalk에 쌓인 로그를 분석해,  
  **어떤 문제가 있었는지 / 어떤 시도를 했고 왜 실패했는지 / `Resolved`로 확정된 해결은 무엇이며 어떤 근거로 해결되었는지**를 정리한다
- `Resolved` 처리된 해결 방법만을 기준으로 요약해, 검증되지 않은 시도와 명확히 구분한다
- 정리된 내용을 바탕으로 **트러블슈팅 중심의 기술 블로그 초안**을 생성해, 기록을 하나의 자산으로 만든다
- 추가 기능으로, 로그를 보고 **부족해 보이는 개념이나 더 학습하면 좋은 주제**를 함께 추천한다

→ 흩어진 개발 로그 중 **검증된 해결 과정만 정리해 글로 남겨 하나의 자산으로 만드는 서비스**
